# Community patterns

Shared taste patterns live at:

```text
community/patterns/<category>/<name>.json
```

Each file is exactly one `ProfileV1.taste.patterns[]` entry, the same shape
published in a profile gist. Use `ax contribute pattern` to open the fork PR:
it can pick from your local profile's taste patterns, or `--fresh` can guide
you through the schema fields. There is no blank-editor path.

## Categories

`category` is closed:

- `design-aesthetic`
- `problem-solving-strategy`
- `debugging`
- `failure-mode`
- `workflow`
- `tool-output-mix`
- `stack-choice`

The directory must match `category`; the filename stem must match `name`.
Evidence is required on every pattern. A PR that modifies an already-existing
`community/patterns/<category>/<name>.json` fails validation, which forces the
contributor to extend or link the existing pattern instead of duplicating it.

## Schema

Prose categories require:

```json
{
  "category": "workflow",
  "name": "small-review-loops",
  "summary": "Review the diff in small increments before expanding scope.",
  "evidence": { "sessions": 4, "confidence": 0.8 }
}
```

`stack-choice` requires `slot` instead of `summary`:

```json
{
  "category": "stack-choice",
  "slot": "state-management",
  "name": "effect-atom",
  "over": ["redux", "zustand"],
  "context": "react apps",
  "evidence": { "sessions": 5, "confidence": 0.9 }
}
```

Optional `links[]` entries use `rel` of `recovered-by`, `pairs-with`, or
`conflicts-with`, and `ref` as `<category>/<name>`.

## Generated files

`seed.json` is the legacy curated starter pack for the public patterns page.
`trending.json` is generated by the community compiler when enough
contributed patterns exist. Do not edit generated files by hand.
