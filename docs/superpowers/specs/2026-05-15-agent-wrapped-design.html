Product shape

Two outputs, one profile.

Agent Wrapped turns the local `ax` evidence graph into an internal report and a sanitized public story deck. The report is the trusted surface. The deck is the shareable hook.

The core question is: What kind of agent collaborator am I?

Outputs

Internal report

Local `/wrapped` route with a Claude-style usage snapshot, archetypes, evidence, sessions, tool calls, skills, failures, verification behavior, and links into existing dashboard pages.

Public deck

X-friendly portrait cards with aggregate facts, including one compact usage card and meme-readable facts like Token Maxxing, redacted by default and ready for screenshots or static export.

Usage snapshot

Borrow Claude Code's overview pattern.

The internal report should lead with compact metrics people already understand: sessions, messages, total tokens, active days, current streak, longest streak, peak hour, favorite model, and an activity heatmap.

The public deck can include a sanitized "usage wrapped" card with aggregate counts and one memorable scale comparison, such as tokens compared with a familiar book. No prompts, paths, repo names, command text, or raw transcript excerpts are needed.

Personality lens

Deterministic archetypes from graph behavior.

V1 should use explainable scoring rules. A card should never claim an archetype unless enough graph evidence exists.

The VerifierHigh verification commands, completion checks, test usage, and low unverified completion rate.
The DebuggerHigh friction-to-recovery activity, diagnostic loops, and repeated failed-command recovery.
The OrchestratorPlans, subagents, tool diversity, and cross-session coordination.
The Skill CollectorSkill diversity, skill pairing, and active use of installed skills.
The RefactorerFile-change breadth, architecture notes, refactor commits, and co-change clusters.
The Context CuratorRecall, file-context, project-context, and grounding before edits.
The Tool TamerBroad tool usage with declining failures or strong recovery from tool failures.
Interesting facts

Make the graph sound like the internet.

Wrapped needs small facts with memorable labels, not dry metric names. Each fact should have internal evidence, public-safe copy, and a deterministic query rule.

Token MaxxingBiggest token day, token-heavy sessions, token streaks, and scale comparisons.
Tool TamerMost-used tool, tool diversity, and strongest recovery from tool failures.
VerifycelUnusually high test, check, or project verify activity before completion.
Context GobblerHigh recall, context, file-read, or search volume before edits.
Skill StackerFrequent skill pairings or unusually diverse skill invocation.
Subagent SummonerFrequent spawned-agent or orchestration behavior.
Repo Monogamist / Repo HopperDepth in one repository versus breadth across many repositories.
Night Shift BuilderSessions clustered late at night or a clear peak-hour pattern.
Internal profile

Rich and local.

  • May include sensitive labels and links.
  • May include local-only evidence.
  • Can deep-link to sessions, tools, skills, projects, workflow, and recall.
  • Never leaves the machine unless the user exports or publishes it.
Public profile

Aggregate by default.

  • No repository names.
  • No file paths.
  • No command text or raw error text.
  • No prompt or transcript snippets.
  • No usernames, local paths, remotes, or private issue references.
Data contract

`WrappedProfile` sits below rendering.

Query code computes the profile. Dashboard code renders it. Export code sanitizes it. Future static/R2 publishing consumes only the sanitized artifact bundle.

interface WrappedProfile {
  readonly period: {
    readonly label: string;
    readonly startedAt: string;
    readonly endedAt: string;
  };
  readonly primaryArchetype: WrappedArchetype;
  readonly secondaryArchetypes: readonly WrappedArchetype[];
  readonly facts: readonly WrappedFact[];
  readonly metrics: WrappedMetrics;
  readonly privacy: WrappedPrivacySummary;
}
Flow

Dashboard

  1. Open `/wrapped`.
  2. Fetch `/api/wrapped`.
  3. Render archetype, traits, facts, and evidence.
  4. Preview public deck.
  5. Adjust redaction.
  6. Export static bundle.
API

Endpoints

  • GET /api/wrapped?period=year
  • GET /api/wrapped/public-preview?period=year
  • POST /api/wrapped/export
Sources

Graph evidence

`session`, `turn`, `tool_call`, `skill`, `invoked`, `skill_paired`, `friction_event`, `diagnostic_event`, `recovered_by`, `plan`, `repository`, `commit`, `produced`, `touched`, `session_token_usage`, and derived signals.

V1 delivers

Local report and export bundle.

  • `/wrapped` route.
  • Deterministic `WrappedProfile` query.
  • 5-7 archetypes with documented scoring.
  • Claude-style usage overview and activity heatmap.
  • Internal evidence-backed report.
  • Sanitized public preview.
  • Static export bundle to a local directory.
Deferred

Keep publishing out of v1.

  • PNG generation if HTML cards are enough for screenshots.
  • R2 publishing.
  • Public permalink management.
  • Share text generation.
  • Time-series animation or video.
Future static publishing

R2 receives only sanitized artifacts.

  1. Generate `wrapped-public.json`.
  2. Generate static HTML/CSS assets.
  3. Upload to R2 or another static host.
  4. Return a public URL.

The publisher should not need database access and should not receive raw internal evidence.

Tests

Safety and scoring need fixtures.

  • Archetype scoring on fixed fixture profiles.
  • Privacy sanitizer removes sensitive fields.
  • Public profile cannot include repo names, file paths, raw command text, or transcript excerpts by default.
  • Dashboard API handles empty, sparse, and rich graphs.
Open decisions

Still to choose.

  • Exact visual identity for public cards.
  • PNG generation versus browser screenshots.
  • Default period: year-to-date, last 365 days, or all-time.
  • Persist archetype scoring or compute on demand.
