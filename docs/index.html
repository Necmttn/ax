what ax does.

It reflects. Then it earns the change.

Every sub-agent you spawn finishes its work and disappears. Whatever it figured out - which command failed three times before the right one, which file actually mattered, which approach to skip - dies with it. ax closes the loop before the session ends, and proves whether the fix earned its place.

  • retros

    Every session ends with a structured note - tried, worked, failed, next. Main sessions and sub-agents alike. JSON by default.

  • experiments

    Accepted proposals become artifacts you can point at. Each one gets a t+7 / t+30 / t+90 checkpoint.

  • evidence

    Verdicts come from retros, tool calls, corrections, and local git outcomes - joined in one graph. No vibes.

enforcement.

Prose drifts. The hook does not.

A line in CLAUDE.md is a suggestion. The model loses it under context pressure and finds itself back on main, the same correction scattered across weeks of sessions. ax moves the rule down the stack - out of prose, into a tool-time hook the agent cannot skip.

The figure below is the boundary. Watch the agent's tool stream drift right. In prose rule mode every call crosses and incidents accumulate. In tool hook mode the dangerous ones stop at the line. Same intent, different layer.

the loop, end to end.

Retro. Propose. Verdict.

Drag a retro across. Click start experiment. Drive time forward and watch what the session actually earned, three months on. Kept means no regressions. Regressed means the PRs came back. Self-resolved means the pattern stopped showing up before the fix mattered.

install

30 seconds. Then you're indexing.

Requires Bun ≥ 1.3 and SurrealDB ≥ 3.0. macOS-first; Linux works for ingest + CLI.

$ curl -fsSL https://raw.githubusercontent.com/Necmttn/ax/main/install.sh | bash
$ PATH="$HOME/.local/bin:$PATH" axctl ingest --since=7
$ axctl serve   # live dashboard at http://127.0.0.1:8520

For dev setup, schema, and benchmarks, see docs/development.md.
