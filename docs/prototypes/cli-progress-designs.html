agentctl ingest progress concepts

Three terminal-first directions for making long ingest runs feel alive while keeping logs clean. The default should be useful in a normal shell, degrade to plain text in CI, and reserve heavier visuals for explicit modes.

mockup only / no runtime code

1. Compact Default

Best everyday mode. One live block, final summary, no screen takeover.

agentctl ingest --since=7
$ agentctl ingest --since=7

agentctl ingest  run=7f42a9  since=7d

✓ skills       upsert        205 skills      0.4s
✓ commands     upsert         42 commands    0.2s
⠋ claude       transcripts    1,284 sessions  8.1s
• codex        sessions       pending
• git          history        pending
• signals      derive         pending

stage claude/transcripts
read 189,420 turns · 4,211 tool calls · 28 plans

press ctrl-c once to stop after current stage
BehaviorUpdates in place when stdout is a TTY. Prints normal lines when redirected or in CI.
Why it worksSmall enough to trust as the default. Shows exactly where time is going without hiding errors.
ImplementationHook into existing telemetryStage start/finish events. No ingest rewrite.
Recommendation: ship this first as the default interactive progress view.

More visual, still terminal-native. Good for long backfills and demos.

agentctl ingest --since=90 --progress=pipeline

BehaviorShows elapsed time, ETA, rows processed, rows per second, current stage counters, write speed, and memory.
RiskSome stages do not know total work upfront. Those show counters and speed without fake percentages.
ImplementationAdd progress events: planned, started, advanced, written, finished, failed.
Preferred direction: make this the interactive default as --progress=pipeline, with plain logs for non-TTY.

3. Agent Cockpit

A polished “show me what the agent is learning” mode for humans watching runs.

agentctl ingest --watch --progress=cockpit
