## rel-batch-a plan

Tasks:
- Read `docs/releases/README.md` and the newest existing release page for style.
- For each target version (`0.25.0`, `0.30.0`, `0.31.0`, `0.32.0`, `0.33.0`), run `bun run release:announcement -- X.Y.Z` to scaffold evidence.
- Rewrite only the five target files under `docs/releases/` into concise announcement pages grounded in their scaffolded commit/file evidence.
- Add release-page coverage that fails while these pages are missing or structurally incomplete, then make it pass with the real pages.

Files:
- `.plan.md`
- `docs/releases/v0.25.0.md`
- `docs/releases/v0.30.0.md`
- `docs/releases/v0.31.0.md`
- `docs/releases/v0.32.0.md`
- `docs/releases/v0.33.0.md`
- A focused `bun:test` release-docs test file, if an existing docs/release test seam is present.

Tests and gates:
- Red/green focused test for the five target release pages.
- `bun run typecheck`
- `bun run --cwd apps/site build`

Sequence:
1. Inspect release docs conventions and existing tests.
2. Add the smallest release-docs test for required pages and confirm it fails.
3. Scaffold and rewrite one version at a time, rerunning the focused test after each page or batch.
4. Run the full required gates.
5. Commit all changes with the required co-author trailer.
